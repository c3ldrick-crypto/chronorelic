Mort Roguelite & Héritage

Mourir n'est pas une fin — c'est un choix de design

La mort comme mécanique de jeu

Quand votre personnage meurt, votre collection, votre base (Sanctuaire) et vos bonus d'héritage accumulés survivent. Seul le personnage disparaît. Un nouveau naît, potentiellement plus fort.

1. La mort survient

Lors d'un échec de capture avec risque de mort actif. Un écran de résolution s'affiche — la tension est palpable.

2. Choisissez votre héritage

3 bonus sont proposés, générés selon la classe et le niveau de votre personnage décédé. Chaque mort offre des options différentes. Vous en choisissez un seul.

3. Un nouveau personnage naît

Choisissez sa classe. Certains bonus d'héritage octroient un démarrage avantageux : niveau 3, ressources bonus, essences de départ.

4. Les bonus s'accumulent

À chaque génération, votre personnage hérite de tous les bonus choisis précédemment. La mort devient une stratégie à long terme.

Comment ça fonctionne

1️⃣

2️⃣

La Machine génère une minute cible avec un indice d'événement historique.

3️⃣

Vous êtes redirigé vers /play avec la minute pré-chargée. Lancez la capture.

🔧 Correctifs critiques — v2.1

Hotfix : Redis & Gameplay

Trois systèmes entiers étaient silencieusement cassés sans Redis configuré. Tous sont maintenant stockés en base de données — le jeu fonctionne pleinement en dev.

🎯

Instinct de Chasse — CORRIGÉ

Bug critique

Le talent Chasseur "Instinct de Chasse" garantissait RARE minimum... mais donnait systématiquement des COMMUNES. Cause racine : le flag d'activation était stocké dans Redis (non configuré) → no-op silencieux.

Correction : Les flags guaranteedRareNext et rerollBonusNext sont maintenant des champs booléens en base de données sur le Character. Idem pour le compteur d'utilisations d'aptitude journalier (abilityUsedDate + abilityUsesLeft). Fonctionne sans Redis.

🗺️

Expéditions — CORRIGÉ

Bug critique

Cliquer "Entrer" dans une ère déduisait les ressources mais n'ouvrait aucune salle. Cause racine : l'état de l'expédition active (salles, progression, ère) était stocké exclusivement dans Redis → perdu immédiatement.

Correction : L'état complet de l'expédition (activeExpeditionData) et les cooldowns (expeditionCooldowns) sont maintenant des champs JSON en base de données. Les salles Boss garantissent RARE+ et donnent ×2 ressources. Bonus de complétion améliorés (scalent avec le niveau de l'ère).

💀

Mort & Héritage — CORRIGÉ

Bug critique

Quand un personnage mourait, la sélection du bonus d'héritage était impossible (modal vidé au rechargement). Cause racine : les options d'héritage étaient stockées dans Redis avec un TTL d'1 heure.

Correction : L'état de mort en attente (pendingDeathData) est maintenant stocké en DB sur le Character. La mort → sélection d'héritage → nouveau personnage fonctionne désormais même après rechargement de page.

💎

Mises — Garanties de rareté

Équilibrage

Risquer des ressources et obtenir une COMMUNE était frustrant et injuste. Les mises élevées garantissent maintenant un niveau de rareté minimum :

🪙

💀
Rituel de Sang ×8 — RARE garantie + +40% ÉPIQUE + +20% LÉGENDAIRE
